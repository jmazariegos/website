Minesweeper AI

  • Github Repo
  • Website

What is it?

  • A minesweeper game with an AI attached to it!
  • The AI is meant to solve for every possible 3x3 square probability then see if it can use that knowledge to win games

What did I do?

  • I created the entire minesweeper game from scratch in javascript along with the AI, the visuals were all created by me as well
  • The game was done using clickable images where the board is created by the createBoard function which places the amount of tiles specified and then randomly places the amount of bombs for the game but avoiding where it was clicked to avoid losing immediately and the buttons when clicked to a recursive function to check whether the next tiles around are empty and if they are continue checking around them until they hit a tile with a bomb near it which then marks that tile with the number of bombs near it
  • The timer is done using a timer interval same with the placing the flag down with left click but can also be done instantly with right click, technically you can make a board of any size but for limiting (and other reasons) I have limited it to the normal amount, the reset button is the face and will always create the board with the settings that are there
  • The AI on the other hand was made with a reinforcement learning method in mind so its a 3 layer AI with the input layer as the full board, the hidden layer as the 3x3 squares that are on the board, and the output layer as the single tile from the 3x3 square, and the hidden layer is made up of only one version of each 3x3 square so that each board influences the probability of a 3x3 square by making it easier to determine its value, routing to output, and smaller size for AI
  • When the AI starts it adds the current board into the network then adds all new 3x3 squares within the board and attaches any existing 3x3 squares in the network to the board with the corresponding positions of each tile in the 3x3 square to the full board (real position on board) and the connection from the 3x3 square to the tile containing the position of the tile in 3x3 square (relative position)
  • The probabilities start at an equal level for each tile that isn't revealed yet so a new 3x3 square with 3 unrevealed tiles will have each tile with a probability of 1/3 of being a bomb and depending on what the percentage reward is, it will increase tiles that have not been a bomb (while decreasing the others), or decrease them (and vice versa) which happens through a backward pass if the tile is a bomb or not and the forward pass finds the tile with the highest probability
  • Lastly the AI has settings for making it do runs on its own so it can get trained and the delay if you want to watch it, which is still in development for the time being
